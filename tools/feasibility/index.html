How This Agent Works — Real-Time Process
1
You submit a project brief
Enter the project name, location, typology (e.g. BNG housing, office, retail), GLA, unit count, and site conditions. This forms the structured input to the agent — the same information a QS would capture in a project brief meeting.
2
The agent queries its benchmark database
The cost engine immediately queries a live PostgreSQL database of construction benchmark rates — seeded from the ASAQS Building Cost Index and calibrated by typology, specification level, grade, and region. No manual lookup is required.
3
Costs are computed element by element
Building works, bulk services, external works, contingency, and escalation are each calculated using deterministic rules. Professional fees are applied using the SACAP, ECSA, and ASAQS recommended fee scales — automatically, at the correct project value band.
4
A six-layer accuracy gate validates all numbers
Before any report is issued, the agent runs three cross-checks: schema validation (every field in range), rules validation (VAT=15%, fee scales applied correctly), and triangulation (cost computed three independent ways, coefficient of variation ≤10%). Benchmark deviation and range sensitivity are also tested.
5
Claude 3.5 drafts the executive narrative
Once the numbers pass the quality gate, the validated cost data is passed to Anthropic's Claude 3.5 Sonnet model which drafts a professional QS narrative — explaining the cost outcome, viability verdict, and recommended adjustments in plain language for developers, funders, and approval boards.
6
A structured report is returned in seconds
The complete feasibility cost report — with cost build-up, professional fees schedule, market analysis, scale sensitivity, risk register, and accuracy dossier — is rendered on screen and available for Excel export. What a QS team would take 2–3 days to produce manually is delivered in under 30 seconds.
