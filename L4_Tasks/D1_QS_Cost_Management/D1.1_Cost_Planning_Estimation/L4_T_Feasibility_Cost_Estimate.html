1. Task Overview

This document decomposes the Feasibility Cost Estimate sub-process into granular, executable tasks with specific data requirements, decision criteria, quality checks, and tools. Each task represents a discrete unit of work that can be assigned, tracked, and verified.

2. Detailed Task Breakdown

TASK 1.1.1-01

Receive & Scope Project Brief

Description
Receive the developer/client brief. Confirm project parameters: unit count, typology, site location, target completion date, funding model. Log the request and assign an estimate reference number.
Role
QS (Lead), Developer (Input)
Inputs
Developer brief document, site coordinates/erf number, unit count & mix, concept sketch (if available)
Outputs
Scoped estimate request form, estimate reference number
Tools / Templates
Estimate Request Template (ERT-001), Project Registry
SLA
Within 4 hours of receipt

✓ Quality Check: All mandatory fields on request form completed; project registered in system

TASK 1.1.1-02

Collect Site & Market Data

Description
Gather site-specific data: geotechnical conditions (preliminary), bulk services availability, municipal development charges schedule, zoning restrictions. Collect market benchmarks: BER Building Cost Index, Rode's Report, ASAQS cost data, internal portfolio historical data.
Role
QS (Lead), Municipal Liaison
Inputs
ERF number, municipal website/GIS, BER subscription, Rode's database, internal project database
Outputs
Data pack: municipal fees, bulk contribution estimates, geotechnical risk rating, benchmark rates
Tools / Templates
Site Data Collection Checklist (SDC-001), BER Online Portal, Rode Report
SLA
2 working days

✓ Quality Check: Benchmark rates not older than 6 months; municipal fees confirmed (not estimated)

TASK 1.1.1-03

Determine Unit Typology & Specification Level

Description
Define building typology (RDP 40m², BNG 50m², social housing, GAP, open market). Establish specification level: standard subsidy, enhanced subsidy, mid-market, premium. Map to ASAQS cost benchmarks per typology. Confirm with architect's concept sketch if available.
Role
QS + Architect (Joint)
Inputs
Developer requirements, concept layouts, DHS specification norms (for subsidy housing)
Outputs
Typology schedule: unit types, areas, spec level, applicable benchmark range
Decision Criteria
If subsidy project: must comply with DHS Minimum Norms & Standards. If GAP: must fall within FLISP qualification range.
SLA
1 working day (meeting with architect)

✓ Quality Check: Typology matches funding model; areas comply with minimum norms

TASK 1.1.1-04

Calculate Building Works Estimate

Description
Apply benchmark rates per m² (or per unit for standard typologies) to calculate building works cost. Adjust for: location factor, specification level, current market conditions. Calculate per-unit cost × total units. Include communal facilities if applicable (social housing).
Role
QS (Lead)
Inputs
Typology schedule, benchmark rates (BER/ASAQS/internal), unit count & mix
Outputs
Building works estimate (subtotal), rate per m², rate per unit
Tools / Templates
Feasibility Estimate Spreadsheet (FES-001), BER Cost Model
SLA
1 working day

✓ Quality Check: Rate/m² falls within ±10% of published ASAQS benchmark for typology

TASK 1.1.1-05

Calculate External Works & Bulk Services

Description
Estimate external works: roads, stormwater, sewer, water reticulation, electrical reticulation, landscaping. Estimate bulk services contributions: sewer, water, roads, electricity connection fees per municipal schedule. Add land cost / acquisition cost if applicable.
Role
QS (Lead), Civil Engineer (Consult)
Inputs
Municipal development charges schedule, site development plan (preliminary), civil engineer input (if available)
Outputs
External works estimate, bulk contributions schedule, infrastructure cost subtotal
Decision Criteria
If bulk services exceed 20% of building cost → flag for client review. If off-grid solutions needed → obtain specialist quote.
SLA
1 working day

✓ Quality Check: Municipal fees from current published schedule; infrastructure allowance realistic for site conditions

TASK 1.1.1-06

Add Professional Fees, Contingencies & Escalation

Description
Calculate professional fees: Architect (SACAP guideline %), QS (SACQSP guideline %), Engineers (ECSA guideline %), PM fees. Add contingency: 10% design development, 5% construction. Calculate escalation to mid-point of construction using BER index projections.
Role
QS (Lead)
Inputs
Works cost subtotal, SACAP/SACQSP/ECSA fee scales, BER escalation forecast, project timeline
Outputs
Professional fees schedule, contingency allowance, escalation calculation, total project cost estimate
Calculation Rules
Professional fees: 12-18% of works cost (varies by project size). Contingency: 10-15% at feasibility stage. Escalation: compound to mid-point of construction.
SLA
0.5 working days

✓ Quality Check: Professional fees within gazetted guidelines; escalation rate sourced from published forecast

TASK 1.1.1-07

Compile Feasibility Report & Present

Description
Compile all calculations into a structured feasibility cost report. Include: executive summary, cost breakdown, assumptions register, exclusions list, benchmark comparison, recommendation (viable / viable with conditions / not viable). Present to developer/funder.
Role
QS (Lead), Senior QS (Review)
Inputs
Building works estimate, external works, fees, contingency, escalation, subsidy quantum
Outputs
Feasibility Cost Report (signed), presentation to stakeholders
Decision Criteria
If cost ≤ subsidy quantum + developer contribution → Viable. If cost exceeds by ≤15% → Viable with conditions (recommend VE). If exceeds by >15% → Not Viable.
SLA
1 working day (report); presentation within 3 days

✓ Quality Check: Report reviewed by Senior QS; all assumptions documented; accuracy range stated (±15-25%)
