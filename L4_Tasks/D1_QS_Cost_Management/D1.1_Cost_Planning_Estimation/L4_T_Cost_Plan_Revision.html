1. Task Overview

Decomposes the Cost Plan Revision sub-process into tasks for maintaining the cost plan as a living document through design changes, scope updates, and market fluctuations. Each revision is version-controlled with delta reporting.

2. Detailed Task Breakdown

TASK 1.1.3-01

Receive Design Change Notification

Description
Receive formal design change notification or scope update from architect/engineer/client. Log change in the project change register with date, originator, description, and affected disciplines. Categorise as: minor (finishes/fittings), moderate (layout/structural), or major (scope/programme impact).
Role
QS (Lead)
Inputs
Revised drawings, architect instruction (AI), scope change request, client directive
Outputs
Change register entry, change impact categorisation
Tools / Templates
Change Register Template (CRT-001)
SLA
Day of receipt

✓ Quality Check: Change logged with unique reference number; categorisation confirmed

TASK 1.1.3-02

Identify Affected ASAQS Elements

Description
Map the design change to specific ASAQS elements affected. Cross-reference with current baseline cost plan. Determine if change affects quantities only, rates only, or both. Identify secondary impacts (e.g., structural change may also affect services routing and external works).
Role
QS (Lead)
Inputs
Change documentation, current baseline cost plan, ASAQS element structure
Outputs
Affected elements list with primary and secondary impacts
Decision Criteria
If change affects >3 elements → classify as major revision. If single element only → minor revision. Major revisions require Principal QS involvement.
SLA
1 working day

✓ Quality Check: All primary and secondary element impacts identified; no orphan impacts

TASK 1.1.3-03

Re-measure Quantities for Affected Elements

Description
Re-measure quantities for each affected element using revised drawings. Document both previous and revised quantities to enable delta comparison. For partial changes, isolate the changed portion from unchanged quantities. Maintain measurement audit trail.
Role
QS / Cost Technician
Inputs
Revised drawings, previous quantity schedule, affected elements list
Outputs
Updated quantity schedule showing previous / revised / delta per element
Tools / Templates
Elemental Take-off Template (ETO-001), CostX (if BIM), red-line overlay comparison
SLA
2–5 working days (proportional to change scope)

✓ Quality Check: Delta quantities clearly documented; measurement audit trail complete

TASK 1.1.3-04

Re-price with Current Market Rates

Description
Apply current rates to revised quantities. For unchanged items, check if rates need escalation adjustment (if >6 months since last pricing). For new items (spec changes), source new rates from benchmark databases or quantity enquiries. Document rate adjustments with justification.
Role
QS (Lead)
Inputs
Updated quantities, current rate database, BER escalation index, new specifications
Outputs
Revised priced element schedule with rate change commentary
Decision Criteria
If BER index moved >3% since last baseline → escalate all rates. If new specification item → obtain minimum 2 supplier quotes or use published rates.
SLA
1–2 working days

✓ Quality Check: Rate adjustments justified; escalation applied consistently across all elements

TASK 1.1.3-05

Update Contingency Assessment

Description
Reassess contingency levels based on current design maturity. As design progresses, contingency should reduce (design development contingency decreases; construction contingency remains). Update risk register items that have been resolved or emerged. Recalculate contingency amount.
Role
QS (Lead)
Inputs
Risk register, design completion percentage, works subtotal
Outputs
Updated contingency schedule with justification
Calculation Rules
Design contingency: 10% at concept → 7.5% at design dev → 5% at detail design → 2.5% at tender. Construction contingency: maintains at 3–5%.
SLA
1 working day

✓ Quality Check: Contingency reduction justified by design maturity evidence; risk items updated

TASK 1.1.3-06

Compare to Previous Version & Document Variances

Description
Produce element-by-element comparison: previous version vs revised version. Calculate absolute (R) and percentage (%) variance per element. Prepare variance commentary explaining the reason for each movement: design change, rate adjustment, scope change, contingency release, escalation. Flag elements with >5% movement.
Role
QS (Lead)
Inputs
Previous baseline cost plan, revised cost plan
Outputs
Delta report: element movement schedule with commentary, overall budget position
Tools / Templates
Cost Plan Comparison Template (CPC-001)
SLA
1 working day

✓ Quality Check: Every element with >5% variance has documented reason; net position reconciled

TASK 1.1.3-07

Issue Revised Cost Plan & Update Assumptions Register

Description
Compile revised cost plan with delta report. Update assumptions register: remove assumptions resolved by design progress, add new assumptions for areas still undefined. Senior QS review and sign-off. Register new version in document management system (increment version number). Distribute to project team with covering note highlighting key changes.
Role
Principal QS (Approval), QS (Preparation)
Inputs
Revised cost plan, delta report, updated assumptions register
Outputs
Versioned cost plan (vN.x), distribution register, covering note
Tools / Templates
Cost Plan Report Template (CPR-001), Document Management System
SLA
1 working day

✓ Quality Check: Version number incremented correctly; assumptions register current; Senior QS signed
