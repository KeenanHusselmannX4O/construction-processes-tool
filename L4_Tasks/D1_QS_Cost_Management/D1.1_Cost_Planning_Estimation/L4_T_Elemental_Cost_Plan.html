1. Task Overview

Decomposes the Elemental Cost Plan sub-process into granular tasks covering quantity extraction per ASAQS elements, rate application, budget comparison, and value engineering. Accuracy target: ±10–15% (ASAQS Class 3).

2. Detailed Task Breakdown

TASK 1.1.2-01

Receive & Review Design Development Drawings

Description
Receive architectural, structural, and services design development drawings. Compile drawing register. Confirm design stage (RIBA Stage 3 / SACAP Stage 3 equivalent). Identify scope gaps or missing information and issue RFIs to design team.
Role
QS (Lead)
Inputs
Architectural plans/sections/elevations, structural GA drawings, services layouts, door & window schedules, finishes schedule
Outputs
Drawing register, RFI list (if any), scope confirmation memo
Tools / Templates
Drawing Register Template (DRT-001), RFI Log
SLA
1 working day

✓ Quality Check: All discipline drawings received and cross-referenced; design stage confirmed

TASK 1.1.2-02

Extract Quantities per ASAQS Elemental Format

Description
Measure and extract quantities for each ASAQS standard element: (1) Substructure — foundation areas, depths, ground slab area; (2) Superstructure — wall areas, frame quantities, roof area, stair count; (3) Finishes — wall/floor/ceiling areas per finish type; (4) Fittings — schedule from architect; (5) Services — from engineer's layouts; (6) External Works — from civil/landscape drawings.
Role
QS / Cost Technician
Inputs
Design drawings, schedules, specifications, BIM model (if available)
Outputs
Quantity schedule per ASAQS element (m², m³, nr, item)
Tools / Templates
Elemental Take-off Template (ETO-001), CostX / Buildsoft (if BIM), manual scaling tools
SLA
5–10 working days (depending on project size)

✓ Quality Check: Cross-check total GFA against architect's area schedule (±2% tolerance)

TASK 1.1.2-03

Apply Rates from Benchmark & Tender Databases

Description
Select appropriate rates for each element from: ASAQS published indices, internal tender database (recent projects), BER Building Cost Index. Adjust rates for: location (regional index), market conditions (supply/demand), specification level. Apply rates to quantities to produce priced element schedule.
Role
QS (Lead)
Inputs
Quantity schedule, ASAQS rate indices, internal rate library, BER indices
Outputs
Priced elemental schedule with rate source notation
Decision Criteria
Rate source preference: (1) Recent tender from similar project ≤12 months, (2) ASAQS published index, (3) BER benchmark + adjustment. If rate variance >15% between sources → use average and flag uncertainty.
SLA
3–5 working days

✓ Quality Check: Rate sources documented per element; no rate older than 12 months without escalation

TASK 1.1.2-04

Add Preliminaries & Contingencies

Description
Calculate preliminaries: time-related costs (site establishment, supervision, plant) based on programme duration; fixed charges (insurance, bonds, testing); project-specific items (community liaison, safety officer for mining belt sites). Calculate contingencies: design development contingency (reduces as design matures, typically 5–10%); construction contingency (3–5%).
Role
QS (Lead)
Inputs
Works subtotal, construction programme, contract conditions, risk register
Outputs
Preliminaries build-up, contingency schedule, all-in cost plan total
Calculation Rules
Preliminaries: 10–18% of works cost (build from first principles preferred over %). Contingency: 5% if design 80%+ complete; 10% if design 50–80% complete.
SLA
2 working days

✓ Quality Check: Preliminaries built from first principles; contingency % justified by design maturity

TASK 1.1.2-05

Compare to Feasibility Estimate & Approved Budget

Description
Produce element-by-element comparison between current cost plan and: (a) Original feasibility estimate; (b) Approved client budget. Highlight variances >5% per element. Calculate overall budget variance. Prepare variance commentary for each element exceeding threshold.
Role
QS (Lead)
Inputs
Feasibility cost report, client budget approval, current elemental cost plan
Outputs
Variance analysis report, element movement schedule
Decision Criteria
If total within budget → proceed to issue. If 0–10% over → identify VE options. If >10% over → mandatory VE workshop before issue.
SLA
1 working day

✓ Quality Check: Every variance >5% has documented commentary; overall budget position clear

TASK 1.1.2-06

Value Engineering Workshop (If Over Budget)

Description
Convene design team (architect, engineers, QS). Present element-by-element overruns. Identify savings options per element without compromising: structural integrity, minimum norms compliance, functionality. Document agreed savings with responsible party and deadline for design revision.
Role
QS (Facilitator), Architect, Engineers
Inputs
Cost plan with variance report, element breakdowns, specification details
Outputs
VE savings schedule, revised specifications, action list with deadlines
Decision Criteria
Savings must bring total within approved budget. If impossible → escalate to client with options: increase budget, reduce scope, or defer elements.
SLA
Workshop within 5 working days; revised plan within 10 working days

✓ Quality Check: VE savings do not compromise NHBRC structural requirements or DHS minimum norms

TASK 1.1.2-07

Issue Cost Plan & Register as Version Baseline

Description
Compile final elemental cost plan document. Include: cost summary page, elemental breakdown, assumptions register, exclusions, benchmark comparison. Senior QS review and sign-off. Register in version control system as baseline (v1.0 for initial, v2.0+ for revisions). Distribute to project team and client.
Role
Principal QS (Approval), QS (Preparation)
Inputs
Final cost plan, VE outcomes (if applicable), assumptions register
Outputs
Baselined Elemental Cost Plan (signed), distribution register
Tools / Templates
Cost Plan Report Template (CPR-001), Document Management System
SLA
1 working day

✓ Quality Check: Senior QS signature; all assumptions documented; version registered in DMS
