1. Task Overview

This task set expands the BoQ Compilation & Formatting sub-process (D1.2-SP-002) into granular work instructions. It covers the transformation of signed-off dimension sheets into a structured, tender-ready Bill of Quantities document following ASAQS conventions. The BoQ serves as the pricing document for tenderers and the contractual basis for interim payments under JBCC, NEC4, or GCC 2015 contract forms.

2. Detailed Task Breakdown

TASK 1.2.2-01

Organise Measured Items by ASAQS Trade Section

Receive signed-off dimension sheets from D1.2-SP-001 and sort all measured items into the standard ASAQS trade section order. Create a master item list per section: Earthworks, Concrete & Formwork, Masonry, Waterproofing, Roof Covering, Carpentry & Joinery, Metalwork, Plastering, Tiling, Glazing, Painting, Plumbing, Electrical, External Works. Group related items within each section (e.g., all concrete elements by mix design, all masonry by wall thickness). Assign sequential item numbers within each section.

QS / Cost Technician

Sectioned item list with sequential numbering per trade

✓ Quality Check: All dimension sheet items accounted for in sectioned list; no orphan quantities; item numbering is consecutive

TASK 1.2.2-02

Draft Item Descriptions per ASAQS Standard Wording

Write BoQ item descriptions using ASAQS standard terminology and structure. Each description must include: material specification, workmanship standard, location/application, and unit of measurement. Reference the project specification for material grades (e.g., "25 MPa concrete to suspended slabs" rather than generic "concrete"). Apply ASAQS conventions for composite items (e.g., brickwork descriptions include mortar class, bond pattern, jointing). Ensure descriptions are unambiguous and sufficiently detailed for tenderers to price accurately.

TASK 1.2.2-03

Insert Provisional Sums & Prime Cost Items

Identify all work items that cannot be fully measured from drawings and insert appropriate allowances. Defined Provisional Sums: work that is defined but not yet designed in detail (contractor adds only profit). Undefined Provisional Sums: work not yet defined (contractor adds full preliminaries, profit, and attendance). Prime Cost (PC) Items: nominated supplier/sub-contractor allowances for specified materials or specialist work (e.g., PC Sum for sanitary ware at R 45,000). Include contractor's attendance on each PC item. Coordinate values with design team and verify against project budget estimates.

TASK 1.2.2-04

Compile Summary Pages & Section Collections

Create the BoQ summary and collection pages that aggregate section totals into the contract sum. For each trade section: insert a section total carried forward to the summary. Summary page lists all sections with totals, sub-totals for building works, external works, and services. Include: Preliminaries total (from D1.2-SP-003), contingency allowance (typically 5–10%), and escalation clause provisions where applicable. Final collection page produces the single tender price (excluding VAT) and a separate VAT line.

Decision Criteria
Contingency: 5% for well-defined projects, 10% for projects with significant unknowns; escalation clause only if contract period >12 months

✓ Quality Check: All section totals carry correctly to summary; arithmetic verified; VAT calculated at current rate (15%); no missing sections

TASK 1.2.2-05

Quality Check: Cross-Reference Quantities to Drawings

Senior QS performs independent quality assurance review of the complete BoQ draft. Cross-reference a sample of quantities back to the original dimension sheets and drawings. Verify: item descriptions match specifications, units are correct per ASAQS, quantities are reasonable for project scale, all drawing revisions are current, and no items are duplicated or omitted. Use the "top 20 by value" method — verify the 20 highest-value items first (typically 80% of contract value). Document all findings and corrections required.

Senior QS

Inputs
Full BoQ draft; original dimension sheets; construction drawings (current revision); project specifications
Outputs
QA review report with corrections list; amended BoQ incorporating corrections
Tools / Templates

Decision Criteria
Critical errors (wrong unit, missing section) → must correct before issue; minor formatting issues → correct in final formatting pass

✓ Quality Check: QA report signed by Senior QS; all critical corrections applied and verified; top-20 items validated against drawings

TASK 1.2.2-06

Format for Tender Issue (PDF + Excel Pricing Schedule)

Prepare the final tender-ready BoQ package. Generate two versions: (1) PDF — formatted with consistent headers, page numbers, trade section dividers, and firm branding for the formal tender document; (2) Excel Pricing Schedule — unlocked cells for tenderer rate and amount entry, locked cells for descriptions and quantities, with formula-driven section totals and summary collection. Include a cover page with project name, employer details, QS firm, tender closing date, and return instructions. Compile into the tender document set alongside tender conditions and contract forms.
