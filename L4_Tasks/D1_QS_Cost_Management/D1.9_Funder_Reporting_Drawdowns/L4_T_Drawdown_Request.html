1. Task Overview

This task set expands the Drawdown Request & Certification sub-process into granular work instructions. Each task covers a critical step in the drawdown cycle: verifying conditions precedent (CPs), preparing the drawdown request on the funder's prescribed form, collecting and packaging supporting documentation (QS payment certificate, tax clearance, progress photos, insurance), QS certification of work-in-place, obtaining borrower signature, submitting to the funder via the prescribed channel, and updating the drawdown register upon fund release. Delayed drawdowns cause contractor payment delays and trigger contractual interest claims (JBCC clause 33).

2. Detailed Task Breakdown

TASK 1.9.2-01

Verify Conditions Precedent (CPs) Are Satisfied

Before preparing any drawdown request, verify that all conditions precedent in the funder's loan agreement have been satisfied. CPs are mandatory prerequisites — a drawdown submitted with unsatisfied CPs will be rejected, causing delays. Maintain a live CP checklist that is updated with each drawdown cycle.

Role
QS / PM
SLA
1 working day
Inputs
Loan agreement CP schedule; previous drawdown CP checklist; document repository; insurance broker confirmations
Outputs
Completed CP checklist (all items verified); list of expired/missing CPs requiring resolution
Tools
CP checklist template; document management system; platform compliance module
Standard
Funder loan agreement; SA FICA regulations; banking compliance requirements
  • Work Instruction: Extract the full CP schedule from the loan agreement — typical CPs include: building plan approval, NHBRC enrolment (housing), insurance certificates, contractor tax clearance, site access, and municipal compliance
  • For each CP: verify current status — check expiry dates on certificates, confirm approvals are still valid, and verify document availability
  • Check the contractor's SARS Tax Clearance Certificate — must be valid (within 12 months). If expired, request an updated certificate from the contractor immediately
  • Verify insurance: CAR (Contractor's All Risk) and Public Liability policies are current and cover the drawdown period — obtain confirmation from the broker
  • Flag any expired or missing CPs — these must be resolved before the drawdown request can be submitted
  • If CPs cannot be met (e.g., pending municipal approval), escalate to the PM and developer to determine if a waiver can be obtained from the funder
  • ✓ Quality Check: All CPs verified; expired items flagged; resolution plan for any outstanding CPs; checklist signed and dated
TASK 1.9.2-02

Prepare Drawdown Request on Funder's Prescribed Form

Complete the funder's drawdown request form with the drawdown amount, purpose, and relevant project details. The drawdown amount must match the latest QS payment certificate value (or portion thereof as agreed with the funder).

Role
Quantity Surveyor
SLA
1 working day
Inputs
Funder's drawdown form template; payment certificate (D1.5); CP checklist (Task 1.9.2-01); drawdown utilisation schedule
Outputs
Completed drawdown request form; drawdown amount calculation; utilisation schedule update
Tools
Funder drawdown form; payment certificate data; platform drawdown module
Standard
Funder loan agreement (drawdown mechanics); utilisation period per facility terms
  • Work Instruction: Obtain the funder's prescribed drawdown request form — use the exact version specified by the funder (do not modify the form structure)
  • Enter the drawdown number (sequential), date, and project reference per the funder's filing system
  • Calculate the drawdown amount: typically equals the net payment certificate amount for the current valuation period
  • Where the funder requires a cost category breakdown: allocate the drawdown amount across the approved budget categories per the financial model
  • Update the utilisation schedule: cumulative drawn + this drawdown = new cumulative total; remaining facility = total facility − new cumulative
  • Verify the drawdown does not exceed the facility limit or any per-drawdown caps specified in the loan agreement
  • ✓ Quality Check: Form completed per funder template; amount matches payment certificate; utilisation schedule updated; within facility limits
TASK 1.9.2-03

Collect & Package Supporting Documentation

Assemble all supporting documents required by the funder's drawdown checklist. Typical requirements include: QS payment certificate, contractor tax clearance, progress photographs, programme status, insurance confirmation, municipal approvals, and the drawdown utilisation schedule. Submit a complete pack — incomplete packs are returned, causing delays.

Role
QS + PM
SLA
2–3 working days
Inputs
Funder's drawdown document checklist; QS payment certificate; contractor certificates; site photographs; PM programme report
Outputs
Complete drawdown document pack (items 1–7 per funder requirements); document index; completeness confirmation
Tools
Document checklist; project DMS; platform document assembly module
Standard
Funder drawdown requirements; SARS compliance; SA FICA; professional documentation standards
  • Work Instruction: Use the funder's drawdown document checklist as the master — collect each item in order:
    • 1. QS Payment Certificate (from D1.5-SP-002) — signed original
    • 2. Contractor's Tax Clearance Certificate (SARS) — verify validity date
    • 3. Progress photographs — dated and geo-tagged, showing current state per major trade
    • 4. Programme status report from PM — milestones, delays, projected completion
    • 5. Insurance confirmation — current CAR and Public Liability certificates from broker
    • 6. Municipal / statutory approvals — building plan approval, NHBRC enrolment (if applicable)
    • 7. Drawdown utilisation schedule — updated per Task 1.9.2-02
  • Prepare a document index listing each item, date, and page count for the funder's reference
  • If any document is unavailable: note the expected date and assess whether to delay submission or submit with an undertaking to provide
  • Package all documents in the funder's preferred format (PDF bundle, hard copy, or portal upload)
  • ✓ Quality Check: All 7 document items collected; validity dates current; document index prepared; pack complete for submission
TASK 1.9.2-04

QS Certification: Certify Amount Corresponds to Work-in-Place

Issue a formal QS certification confirming that the drawdown amount corresponds to the value of work completed and in place on site. This is a professional undertaking by the QS and carries professional liability. The certification must be based on a recent site inspection.

Role
QS (Principal / Registered Professional)
SLA
1 working day
Inputs
Payment certificate; site measurement records (D1.5-SP-001); recent site inspection notes; valuation calculation
Outputs
Signed QS certification letter; professional representation to funder
Tools
QS certification template; practice letterhead; professional registration details
Standard
SACQSP Professional Code of Conduct; professional indemnity insurance requirements; funder certification format
  • Work Instruction: Verify that a site inspection has been conducted within the current valuation period — the certification must be based on current site conditions
  • Confirm the payment certificate amount represents work actually completed and in place — cross-reference against the site measurement data
  • Draft the QS certification letter on practice letterhead containing: project name and reference, drawdown number, certified amount (R), date of last site inspection, and the professional statement
  • Professional statement: "I hereby certify that the amount of R[X] represents the value of work completed and in place on site as at [date], based on my inspection on [date], and is a fair and accurate assessment of the contractor's entitlement for this valuation period"
  • Include the QS's professional registration number (SACQSP), practice name, and contact details
  • Sign the certification — this activates professional indemnity liability for the representation made
  • ✓ Quality Check: Certification based on recent site inspection; amount matches payment certificate; professional statement accurate; signed by registered QS
TASK 1.9.2-05

Obtain Developer / Borrower Signature on Drawdown Request

Present the complete drawdown pack to the developer or borrower (the entity that signed the loan agreement) for review and signature. The borrower's signature is required before the funder will process the drawdown. Ensure the authorised signatory signs per the loan agreement mandate.

Role
Developer / Borrower

1–2 working days
Inputs
Complete drawdown pack (Tasks 1.9.2-01 to 1.9.2-04); drawdown request form; signatory mandate from loan agreement
Outputs
Signed drawdown request; borrower's approval; ready for funder submission
Tools
Signatory mandate; electronic signature platform (if accepted by funder); courier service (if hard copy required)
Standard
Loan agreement signatory provisions; company resolution (if corporate borrower); FICA requirements
  • Work Instruction: Present the complete drawdown pack to the developer/borrower with a brief cover summary: drawdown number, amount, and purpose
  • Verify the signatory authority: confirm the person signing is an authorised signatory per the loan agreement (check against the signatory mandate or company resolution)
  • If dual signatories are required (common for corporate borrowers): ensure both parties sign
  • If electronic signatures are accepted by the funder: use the approved e-signature platform for efficiency
  • Record the signing date — the funder may have a validity window from signing to submission
  • Return the signed pack to the QS for submission preparation
  • ✓ Quality Check: Authorised signatory confirmed; signature obtained; signing date recorded; pack returned for submission
TASK 1.9.2-06

Submit Drawdown Request to Funder & Track Approval

Submit the signed and complete drawdown request pack to the funder via the prescribed submission channel. Track the funder's review and approval process. Address any queries promptly to avoid delays in fund release. For housing projects, cash-flow timing is critical — delayed drawdowns cascade into contractor payment delays.

Role
QS / PM
SLA
Same day submission; 5–10 days funder processing
Inputs
Signed drawdown pack (Task 1.9.2-05); funder submission channel details; funder contact information
Outputs
Funder submission receipt; approval status tracking; query response log (if applicable)
Tools
Funder portal / email / courier; submission tracker; platform workflow module
Standard
Funder loan agreement (submission method and processing timeline); JBCC clause 33 (payment timeline implications)
  • Work Instruction: Submit the complete drawdown pack via the funder's prescribed channel — confirm submission requirements (portal, email with specific subject line, or hard copy via courier)
  • Obtain a submission receipt or delivery confirmation — this is evidence of timely submission
  • Log the submission in the drawdown tracker: drawdown number, amount, submission date, expected processing time
  • Follow up after 3 business days if no acknowledgement received — escalate if the contractual processing window is approaching
  • If the funder raises queries: respond within 2 business days with the requested information or clarification
  • Upon funder approval: record the approval date, funds release date, and amount transferred
  • If the funder rejects the drawdown: document the rejection reason, resolve the issue, and resubmit
  • ✓ Quality Check: Submission confirmed; receipt obtained; tracker updated; queries addressed within 2 days; approval/rejection logged
TASK 1.9.2-07

Update Drawdown Register & Reconcile upon Fund Release

Upon confirmation of fund release by the funder, update the drawdown register, reconcile the project account, and confirm that the funds have been received in the designated project bank account. This triggers the contractor payment process per D1.5-SP-003.

Role
Quantity Surveyor
SLA
1 working day after fund release
Inputs
Funder release confirmation; bank statement / receipt; drawdown register; project financial model
Outputs
Updated drawdown register; bank reconciliation; contractor payment instruction trigger; cash-flow forecast update
Tools
Drawdown register; bank reconciliation template; platform financial module; cash-flow model
Standard
Funder loan agreement (fund utilisation); financial management best practice; SACQSP guidelines
  • Work Instruction: Confirm the fund release from the funder — verify the amount matches the approved drawdown request (R0 tolerance)
  • Confirm receipt of funds in the designated project bank account — obtain a bank statement or balance confirmation
  • Update the drawdown register: record the drawdown number, date, amount, cumulative drawn, and remaining facility balance
  • Reconcile: total drawdown received = sum of all drawdown register entries; remaining facility = total facility − cumulative drawn
  • Trigger the contractor payment process per D1.5-SP-003 — notify PM and Employer that funds are available for payment
  • Update the project cash-flow forecast with the actual drawdown receipt date and amount
  • ✓ Quality Check: Register updated; bank receipt confirmed; reconciliation complete; contractor payment triggered; cash-flow forecast current
