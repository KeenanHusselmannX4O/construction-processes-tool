1. Task Overview

This task set expands the Monthly Cost Report for Funder sub-process into granular work instructions. Each task covers a critical step in the monthly funder reporting cycle: gathering financial data from project registers, populating the funder's prescribed template (sections A–H), reconciling budget vs actual vs drawdown utilisation, preparing variance explanations, compiling risk and programme updates, conducting internal QA review, and submitting the report to the funder or DFI by the contractual deadline. Funder reports must reconcile to R0 variance against the latest payment certificate and project financial model.

2. Detailed Task Breakdown

TASK 1.9.1-01

Gather Data from Latest Payment Certificate, PFC & Registers

Collect all source data required for the monthly funder report: the latest payment certificate (D1.5), projected final cost (D1.7), variation register (D1.6), contingency status (D1.7-SP-002), drawdown register, and programme status from the PM. This is the data assembly task that feeds all report sections.

Role
Quantity Surveyor
SLA
Day 1–2 after month-end cutoff
Inputs
Payment certificate (D1.5); PFC (D1.7-SP-001); VO register (D1.6); contingency register; drawdown register; PM programme report
Outputs
Data pack for funder report; checklist of source documents gathered; data gap list (if any)
Tools
Source document checklist; platform data extract; project financial model
Standard
Funder loan agreement (reporting obligations); contractual month-end cutoff date
  • Work Instruction: On the contractual month-end cutoff date, request/obtain the following: latest payment certificate, PFC update, VO register extract, contingency status, drawdown register, and PM programme update
  • Verify that the payment certificate period aligns with the reporting month — if there is a timing mismatch, use the most recent certified values and note the cutoff date discrepancy
  • Extract key figures: cumulative expenditure to date (from payment certs), PFC per cost category, total approved VOs, pending VOs, contingency utilised and remaining
  • Obtain the drawdown register: total facility amount, cumulative drawn, available balance, and forecast next drawdown
  • Request programme status from the PM: key milestones achieved this month, projected completion date, and any delays
  • Complete the data gathering checklist — flag any missing items and follow up immediately
  • ✓ Quality Check: All source documents gathered; figures cross-referenced for internal consistency; data gaps identified and escalated; data pack ready
TASK 1.9.1-02

Populate Funder Reporting Template (Sections A–H)

Transfer the gathered data into the funder's prescribed reporting template. Complete all eight sections: Executive Summary, Budget vs Actual, Projected Final Cost, Drawdown Summary, Variation & Claims, Contingency Status, Programme Status, and Risk Register.

Role
Quantity Surveyor
SLA
Day 3–4
Inputs
Data pack (Task 1.9.1-01); funder's reporting template; previous month's report (for comparison)
Outputs
Draft funder cost report (all sections populated); supporting charts and schedules
Tools
Funder template (Excel/PDF); chart generation tools; platform reporting module
Standard
Funder's prescribed format; DFI reporting standards; accuracy to R0 variance
  • Work Instruction: Open the funder's prescribed reporting template — use the version specified in the loan agreement (do not alter the template structure)
  • Section A (Executive Summary): Draft key financial headlines — total budget, % spent, % complete, PFC, key flags. Complete this section last after all data is populated
  • Section B (Budget vs Actual): Enter line-by-line budget amounts and actual expenditure to date from the payment certificate data; calculate variances
  • Section C (Projected Final Cost): Enter PFC per cost category with trend arrows (up/down/flat) vs previous month
  • Section D (Drawdown Summary): Enter total facility, drawn to date, available balance, and forecast next drawdown amount and date
  • Section E (Variation & Claims): Enter approved VOs (count and value), pending VOs, and claims status
  • Section F (Contingency): Enter original contingency, utilised to date, committed, and remaining balance
  • Section G (Programme): Enter key milestones from PM report; record any delays with revised completion dates
  • Section H (Risk Register): Enter top 5 cost/time risks with probability, financial exposure (R), and mitigation actions
  • ✓ Quality Check: All 8 sections populated; figures match data pack; charts generated; previous month comparison included
TASK 1.9.1-03

Reconcile Budget vs Actual vs Drawdown Utilisation

Perform a three-way reconciliation ensuring the budget figures, actual expenditure, and drawdown utilisation all align. Identify and explain any discrepancies — these are critical for funder confidence and continued drawdown approvals.

Role
QS
SLA
Day 4
Inputs
Populated report (Task 1.9.1-02); project financial model; drawdown register; bank statements (if available)
Outputs
Three-way reconciliation schedule; variance analysis; discrepancy notes
Tools
Reconciliation template; financial model; platform reconciliation module
Standard
R0 tolerance between report and payment certificates; funder reconciliation requirements
  • Work Instruction: Prepare a three-way reconciliation: (1) Approved Budget per funder, (2) Actual Expenditure per payment certificates, (3) Drawdown Utilisation per funder register
  • Check that cumulative expenditure in the report = sum of all payment certificates issued to date — must be R0 variance
  • Check that cumulative drawdown = total funds received from funder — reconcile against bank receipts
  • Calculate the "expenditure vs drawdown" reconciliation: expenditure to date should not exceed total drawn (flag if it does — indicates funding gap)
  • Identify any timing differences (e.g., payment certificate issued but drawdown not yet received) and note them as reconciling items
  • Ensure the remaining facility balance is sufficient to cover projected expenditure to completion — flag any projected funding shortfall
  • ✓ Quality Check: Three-way reconciliation balances to R0; all discrepancies explained; no unexplained funding gaps; projected facility sufficiency confirmed
TASK 1.9.1-04

Prepare Variance Explanations for Outlier Items

For all cost categories where the actual expenditure or PFC varies significantly from budget (typically >5% or >R100,000), prepare clear, concise variance explanation narratives for the funder. Funders require written explanations for material variances.

Role
QS
SLA
Day 5
Inputs
Reconciliation (Task 1.9.1-03); budget vs actual analysis; VO register; contingency utilisation log
Outputs
Variance explanation notes per outlier category; supporting references; recommended actions
Tools
Variance explanation template; cost report narrative section; platform analytics
Standard
Funder's materiality threshold (typically 5% or R100k); professional narrative standards
  • Work Instruction: Identify all cost categories with variance >5% or >R100,000 (whichever is lower per funder requirements) between budget and actual or PFC
  • For each outlier: prepare a structured variance note with: (a) category/item description, (b) budget amount, (c) actual/PFC amount, (d) variance R and %, (e) root cause explanation, (f) corrective action (if required)
  • Common root causes to investigate: VO approvals, re-measurement adjustments, PS/PC adjustments, CPA, programme delays, material price escalation
  • Where the PFC exceeds budget: clearly state whether the overrun is funded (from contingency) or unfunded (risk to funder)
  • Keep narratives concise — funders review many projects; 2–3 sentences per variance is sufficient
  • Cross-reference variance notes to the relevant report section (e.g., "See Section E — VO #12 for R450,000 addition")
  • ✓ Quality Check: All material variances explained; root causes identified; corrective actions stated; narratives concise and cross-referenced
TASK 1.9.1-05

Compile Risk Register & Programme Update

Update the funder risk register with the top 5 cost and time risks facing the project. Incorporate the PM's programme status update including key milestones, delays, and the projected completion date. Funders use this section to assess project delivery risk.

Role
QS + PM
SLA
Day 5
Inputs
Project risk register; PM programme report; previous month's risk assessment; VO and claims pipeline
Outputs
Top 5 risk register (with financial exposure R); programme milestone tracker; projected completion date
Tools
Risk register template; programme bar chart / S-curve; platform risk module
Standard
Funder risk reporting requirements; SACQSP risk management guidelines
  • Work Instruction: Review the project risk register and select the top 5 risks by financial exposure (probability × impact in R)
  • For each risk: state the risk description, probability (High/Medium/Low), financial exposure (R), mitigation action, and status update since last month
  • Typical funder-critical risks: programme delays, contractor capacity, material supply, pending VOs, claims, contingency depletion
  • From the PM's programme report: extract key milestones achieved this month and upcoming milestones for next month
  • State the current projected completion date and compare against the contractual completion date — explain any slippage
  • For housing projects: include the unit completion status — number of units at each construction milestone vs programme target
  • ✓ Quality Check: Top 5 risks updated with current financial exposure; programme status current; completion date projection provided; housing unit count (if applicable)
TASK 1.9.1-06

Internal QA Review & Senior QS Sign-off

Subject the draft funder report to an internal quality assurance review by a senior QS or practice director. The reviewer checks arithmetic accuracy, narrative quality, completeness of sections, and consistency with previous reports. This is a mandatory quality gate before funder submission.

Role
Senior QS / Practice Director
SLA
Day 6–7
Inputs
Draft funder report (all sections); reconciliation schedule; variance notes; previous month's report
Outputs
QA-reviewed report; Senior QS sign-off; review comments log (if revisions needed)
Tools
QA review checklist; report comparison tool; platform QA workflow
Standard
SACQSP quality management standards; practice QA procedures; professional indemnity requirements
  • Work Instruction: The Senior QS reviews the complete draft report against the QA checklist covering: arithmetic accuracy (spot-check 5 line items), narrative coherence, section completeness, and funder template compliance
  • Compare key figures against the previous month's report — verify that movements are logical and explained
  • Check that the Executive Summary (Section A) accurately reflects the detail in Sections B–H — no contradictions
  • Verify the reconciliation (Task 1.9.1-03) is balanced and all discrepancies are explained
  • Check variance notes for professional tone, accuracy, and completeness — funders treat these as formal QS representations
  • If revisions are needed: return to QS with review comments; QS revises and re-submits for final sign-off
  • Senior QS signs the report — this sign-off activates professional liability coverage for the report's contents
  • ✓ Quality Check: QA checklist complete; arithmetic verified; narrative reviewed; Senior QS signed; report ready for submission
TASK 1.9.1-07

Submit Report to Funder & File Submission Record

Submit the approved funder report by the contractual deadline via the funder's prescribed channel (portal upload, email, or hard copy). Log the submission date, method, and recipient for the project record. Follow up to confirm receipt.

Role
QS / PM
SLA
Day 7–10 (per contractual deadline)
Inputs
QA-approved report (Task 1.9.1-06); funder submission requirements; funder portal credentials
Outputs
Submitted report; submission receipt/confirmation; filing record; funder feedback log
Tools
Funder portal / email; document management system; submission tracker
Standard
Funder loan agreement (submission deadline and method); professional correspondence standards
  • Work Instruction: Submit the signed report via the funder's prescribed channel — ensure the submission method complies with the loan agreement requirements
  • If portal submission: upload the report and all supporting schedules; download the upload confirmation receipt
  • If email submission: send to the designated funder representative with the report attached; request a read receipt
  • Log the submission: date, time, method, recipient name, and confirmation reference number in the project submission tracker
  • File a copy of the submitted report (with sign-off) in the project document management system under the reporting month
  • If the funder raises queries: log them in the funder feedback register and respond within the prescribed response time (typically 5 business days)
  • ✓ Quality Check: Report submitted by deadline; receipt confirmed; filed in DMS; submission tracker updated; ready for funder queries
