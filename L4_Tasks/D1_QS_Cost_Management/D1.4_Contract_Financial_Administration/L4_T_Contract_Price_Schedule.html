1. Task Overview

This task set expands the Contract Price Schedule Setup sub-process into granular work instructions. Each task captures the accepted tender amount, agrees the contract sum breakdown by BoQ section, records CPA baseline indices, documents provisional sum and prime cost allocations, establishes retention parameters, builds the master financial tracking register, and obtains formal sign-off. These tasks form the financial foundation upon which all subsequent construction-phase financial administration is built.

2. Detailed Task Breakdown

TASK 1.4.1-01

Record Accepted Tender Sum & Negotiated Adjustments

Capture the accepted tender figure from the letter of acceptance. Cross-reference any negotiated savings, qualifications, or scope adjustments agreed during post-tender negotiations. Record the final agreed contract sum in the contract administration register.

Role
Quantity Surveyor
SLA
1 working day from letter of acceptance
Inputs
Letter of acceptance; post-tender negotiation minutes; tender adjudication report
Outputs
Agreed contract sum recorded; adjustment schedule filed
Tools
Contract register template; platform contract module
Standard
JBCC clause 25 (Contract Sum); NEC clause 50
  • Work Instruction: Open the letter of acceptance and verify the accepted tender figure matches the tender adjudication recommendation
  • Reconcile any post-tender negotiation adjustments (e.g., value engineering savings, programme reductions) against the original tender price
  • Document every adjustment with a line-item reference showing original amount → adjusted amount → reason
  • Record the final agreed contract sum (excl. VAT and incl. VAT) in the contract register
  • ✓ Quality Check: Contract sum reconciles to tender total ± negotiated adjustments; documented approval from PA
TASK 1.4.1-02

Agree Contract Sum Breakdown by BoQ Section

Work with the contractor QS to agree the breakdown of the contract sum by BoQ section. This forms the basis for monthly valuations and progress tracking. Each section total must reconcile to the overall contract sum.

Role
QS + Contractor QS
SLA
2–3 working days
Inputs
Priced BoQ; contract sum from Task 1.4.1-01
Outputs
Signed section-by-section breakdown schedule
Tools
BoQ pricing software; breakdown template spreadsheet
Standard
ASAQS Standard System of Measuring; SACQSP guidelines
  • Work Instruction: Extract section totals from the priced BoQ (Preliminaries, Substructure, Superstructure, Finishes, etc.)
  • Schedule a meeting with the contractor QS to agree the breakdown — confirm each section total and any lump-sum items
  • Ensure the sum of all section totals equals the agreed contract sum from Task 1.4.1-01 (zero variance)
  • Identify sections with Provisional Sums and PC items — mark these for separate tracking
  • Both parties sign the agreed breakdown schedule; file as an annexure to the contract documentation
  • ✓ Quality Check: Section totals sum to contract sum; both QS parties have signed; lump-sum items clearly identified
TASK 1.4.1-03

Record CPA Baseline Indices

Record the Contract Price Adjustment baseline indices at the base month (month preceding tender closing date). These indices are the reference point for all future CPA calculations throughout the contract. Sources include BER Building Cost Index and Stats SA PPI data.

Role
Quantity Surveyor
SLA
0.5 working day
Inputs
BER Building Cost Index publication; Stats SA PPI (steel, fuel, electrical); tender closing date
Outputs
CPA baseline record with all index values and source references
Tools
BER subscription portal; Stats SA website; CPA baseline template
Standard
JBCC CPA provisions; BER methodology
  • Work Instruction: Identify the base month — this is the calendar month immediately preceding the tender closing date
  • Retrieve the BER Building Cost Index value for the base month from the BER publication or subscription service
  • Retrieve Stats SA PPI indices for steel (SIC 2710), fuel, and electrical materials for the same base month
  • Record all index values with their publication reference numbers and dates in the CPA baseline template
  • Cross-reference the contract CPA clause to confirm which indices apply (some contracts use BER only; others use a weighted formula)
  • ✓ Quality Check: All indices match published source values; base month correctly identified relative to tender closing
TASK 1.4.1-04

Document Provisional Sum & Prime Cost Allocations

Extract all provisional sums and prime cost items from the BoQ. Classify each as Defined PS, Undefined PS, or PC item per JBCC definitions. Create the PS/PC register that will track expenditure throughout the contract.

Role
Quantity Surveyor
SLA
1 working day
Inputs
Priced BoQ — PS and PC sections; contract conditions
Outputs
PS/PC register with classifications, allowances, and responsible parties
Tools
PS/PC register template; BoQ extraction tool
Standard
JBCC PS clauses (Defined & Undefined); GCC 2015 clause 49
  • Work Instruction: Extract every PS and PC line item from the priced BoQ — note item number, description, and allowance amount
  • Classify each item: Defined Provisional Sum (work described sufficiently), Undefined Provisional Sum (scope unknown), or Prime Cost item
  • Record the contractor's entitlements per classification: profit-only (Defined PS), full mark-up (Undefined PS), attendance (PC)
  • Identify the responsible design team member or nominated supplier for each item
  • Populate the PS/PC register with status columns: Not Instructed → Instructed → Quoted → Agreed → Expended
  • Calculate total PS/PC exposure and compare against contingency allowance
  • ✓ Quality Check: All BoQ PS/PC items captured; classifications align with contract conditions; register totals reconcile to BoQ
TASK 1.4.1-05

Set Up Retention Parameters

Configure retention deduction parameters as specified in the contract data. Record the retention percentage, retention limit, release triggers, and any guarantee substitution provisions per JBCC clause 32 or equivalent NEC/GCC clause.

Role
Quantity Surveyor
SLA
0.5 working day
Inputs
Contract data (JBCC/NEC/GCC); signed contract agreement
Outputs
Retention schedule with parameters; retention tracking register
Tools
Retention parameter template; financial register
Standard
JBCC clause 32; NEC clause 51.3 (option X16); GCC 2015 clause 44
  • Work Instruction: Extract retention parameters from the contract data: retention percentage (typically 5%), retention limit (typically 5% of contract sum)
  • Calculate the maximum retention amount: contract sum × retention limit percentage
  • Record release triggers: first moiety at practical completion (50%), second moiety at final completion (50%)
  • Note any special provisions: retention guarantee substitution allowed (Y/N), NHBRC requirements for housing projects
  • Set up the retention tracking register with columns for each certificate: certified value, retention deducted, cumulative retention, limit remaining
  • ✓ Quality Check: Retention % and limit match contract data; retention register calculates correctly; release triggers documented
TASK 1.4.1-06

Establish Master Financial Tracking Register

Create the comprehensive financial tracking register that consolidates all contract sum components into a single control document. This register is the QS's primary tool for tracking the financial status of the contract from commencement to final account.

Role
Quantity Surveyor
SLA
1 working day
Inputs
All outputs from Tasks 1.4.1-01 to 1.4.1-05
Outputs
Master financial register with all sections linked and baselined
Tools
Financial register template; platform contract finance module
Standard
SACQSP practice note on contract financial management
  • Work Instruction: Structure the register with the following sections: Contract Sum Summary, Section Breakdown, Variations Register, PS/PC Register, CPA Register, Retention Schedule, Payment Certificate Summary
  • Populate the Contract Sum Summary with: original tender sum, negotiated adjustments, agreed contract sum, contingency allowance
  • Link each section to source data: section breakdown from Task 1.4.1-02, CPA baseline from Task 1.4.1-03, PS/PC register from Task 1.4.1-04, retention parameters from Task 1.4.1-05
  • Set up formulae for automatic calculation of: adjusted contract sum (original + variations + CPA − contingency used), anticipated final account
  • Create a dashboard summary showing: original contract sum, approved variations to date, CPA to date, anticipated final account, % variance
  • ✓ Quality Check: All registers linked and reconciling; formulae tested with sample data; dashboard summary accurate
TASK 1.4.1-07

Obtain PA & Contractor Sign-Off on Contract Price Schedule

Present the complete contract price schedule to the Principal Agent and Contractor for formal review and sign-off. This signed document forms the financial baseline against which all future adjustments, variations, and claims are measured.

Role
Principal Agent + Contractor
SLA
3–5 working days
Inputs
Complete contract price schedule; master financial register
Outputs
Signed contract price schedule; distribution confirmation
Tools
Document management system; digital signature platform
Decision Criteria
Schedule reconciles to accepted tender; all components accurately recorded; PA and contractor agree on breakdown
  • Work Instruction: Compile the contract price schedule into a formal document with cover page, contents, and all component schedules
  • Issue to the Principal Agent for review — allow 2 working days for review and queries
  • Issue simultaneously to the Contractor QS for independent verification
  • Resolve any discrepancies raised by either party — amend the schedule if corrections are required
  • Obtain signatures from: QS (preparer), PA (approver), Contractor representative (acceptance)
  • Distribute signed copies: original to PA, copy to contractor, copy to employer, QS retains working copy
  • ✓ Quality Check: All three parties have signed; no unresolved queries; distribution register completed
