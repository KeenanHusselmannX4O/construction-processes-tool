1. Task Overview

This task set decomposes the Arithmetic Check & Qualification Review sub-process (D1.3-SP-002) into six granular work instructions. The tasks cover the full tender receipt-to-ranking workflow: registering and opening tenders under controlled conditions, verifying tenderer qualification against statutory requirements (CIDB, SARS, B-BBEE), performing line-by-line arithmetic verification of each priced BoQ, applying correction rules per tender instructions, and producing the corrected ranked tender list for comparative analysis. All tasks comply with CIDB Standard for Uniformity procedures and PPPFA 2017 requirements.

2. Detailed Task Breakdown

TASK 1.3.2-01

Register All Received Tender Submissions

Description
Log every tender submission upon receipt in the official tender register. Record: tenderer name, date and time of receipt, seal integrity status (intact/damaged), delivery method (hand/courier/electronic), and name of person receiving. Tenders received after the closing date/time are marked as late and handled per the tender instructions (typically rejected). Secure all tenders in a locked location until the scheduled opening.
Role
PA / QS (receiving officer)
Inputs
  • Sealed tender submissions (physical or electronic)
  • Tender closing date and time from Volume 1
  • Tenderer invitation list
Outputs
  • Completed tender register (receipt log)
  • Late tender notifications (if applicable)
  • Secured tender submissions
Tools / Templates
Tender Register Template (Excel); Late Tender Notification Letter; Secure document storage / electronic tender platform
SLA
0.5 working day (day of tender close)

✔ Quality Check: All tenders logged with exact receipt timestamps, seal integrity recorded, late tenders identified and separated, register signed by receiving officer

TASK 1.3.2-02

Open Tenders in Presence of Witnesses

Description
Open all valid tender submissions in the presence of at least two independent witnesses (typically PA, Employer representative, and/or tenderer representatives for public-sector projects). Read aloud each tenderer's name and total tender amount. Record the read-out totals in the tender opening register. For public-sector tenders, this process must comply with CIDB open-tender-opening requirements. Ensure no tender details are disclosed beyond the read-out totals at this stage.
Role
PA / QS (opening officer)
Inputs
  • Secured sealed tenders (from TASK 1.3.2-01)
  • Tender opening register template
  • Witness attendance register
Outputs
  • Tender opening register (names + read-out totals)
  • Signed witness attendance register
  • Opened tender documents (secured)
Tools / Templates
Tender Opening Register Template; Witness Attendance Register; CIDB tender opening procedure guide
SLA
1 hour (scheduled opening session)
Decision Criteria
Minimum 2 independent witnesses required; for public-sector tenders, tenderer representatives may attend; only tenderer name and total are read aloud — no detailed pricing disclosed

✔ Quality Check: All tenders opened, read-out totals recorded, witness register signed by all attendees, late/damaged tenders dealt with per tender instructions and recorded

TASK 1.3.2-03

Verify Qualification Documents for Each Tenderer

Description
Systematically verify each tenderer's qualification documents against the mandatory requirements set out in the tender instructions. Check: (1) CIDB registration — correct grading designation and class of works for the contract value; (2) SARS tax compliance — verify TCS PIN on SARS eFiling; (3) B-BBEE certificate or sworn affidavit — confirm validity and level; (4) Returnable schedules — all completed, signed, and witnessed where required; (5) JV agreement and CIDB JV registration (if applicable). Record compliance status in the qualification matrix. Flag all non-compliant tenders and notify PA for disqualification decision.
Role
QS
Inputs
  • Tenderer returnable schedules and certificates
  • CIDB Register of Projects (online verification)
  • SARS TCS PIN verification system
  • B-BBEE verification agency database
  • Tender instruction qualification requirements
Outputs
  • Qualification compliance matrix (per tenderer)
  • Non-compliance flags with reasons
  • Recommended disqualifications (for PA decision)
Tools / Templates
Qualification Compliance Matrix Template; CIDB online register; SARS eFiling TCS verification; B-BBEE verification portal
SLA
1–2 working days
Decision Criteria
CIDB grading / tax non-compliance = mandatory disqualification; missing B-BBEE = scored as Level 8+ (non-contributor); incomplete returnables = per tender instruction provisions (may allow clarification or may disqualify)

✔ Quality Check: Every qualification criterion checked for every tenderer, CIDB and SARS verified online (not only from submitted documents), matrix completed and signed by QS

TASK 1.3.2-04

Perform Line-by-Line Arithmetic Check on Each Priced BoQ

Description
For each qualified tenderer, perform a complete line-by-line arithmetic verification of the priced BoQ. Check: (1) rate × quantity = amount for every measured item; (2) page/section sub-totals are correct; (3) carry-forwards between pages are accurate; (4) trade summaries match item totals; (5) Preliminary items are correctly totalled; (6) Summary page correctly aggregates all trades, Prelims, PS, PC, Day Works; (7) VAT calculation is correct; (8) final tender total matches read-out total from opening. Log every error found with item reference, nature of error, original amount, and corrected amount.
Role
QS / Cost Technician
Inputs
  • Priced BoQ from each qualified tenderer
  • Blank BoQ (for reference quantities)
  • Arithmetic Check Schedule template
Outputs
  • Arithmetic check schedule per tenderer (item-level)
  • Error log with original vs corrected amounts
  • Corrected section sub-totals and tender total
Tools / Templates
Arithmetic Check Schedule Template (Excel); BoQ verification spreadsheet with formulae; Calculator / QS software (CostX, QSPlus)
SLA
3–5 working days (depends on number of tenderers and BoQ size)

✔ Quality Check: 100% of line items checked (no sampling), every error logged with item reference, corrected totals independently verified by second QS, check schedules signed off

TASK 1.3.2-05

Calculate Error Percentages & Apply Correction Rules

Description
For each tenderer, calculate the total arithmetic error as a percentage of the original tender total: Error % = (|Corrected Total – Original Total| / Original Total) × 100. Compare the error percentage against the threshold specified in the tender instructions (typically 2%–5%). Determine correction rule application: (a) if error % is within threshold, the corrected total replaces the tendered total for ranking; (b) if error % exceeds threshold, tenderer may be given option to confirm at corrected total or withdraw (per tender instructions). Document the correction decision for each tenderer.
Role
QS
Inputs
  • Arithmetic check schedules (TASK 1.3.2-04)
  • Original tender totals (from opening register)
  • Tender instruction error threshold and correction rules
Outputs
  • Error percentage calculation per tenderer
  • Correction rule application decisions
  • Final corrected tender totals
Tools / Templates
Error Percentage Calculation Sheet; Correction Rule Decision Log; Tender Instructions (correction clause reference)
SLA
1 working day
Decision Criteria
Error within threshold: correct and rank at corrected total. Error exceeds threshold: tenderer confirmation required per tender instructions. Where unit rate and amount differ, unit rate prevails per standard CIDB convention

✔ Quality Check: Error percentages independently verified, correction rule applied consistently to all tenderers, decisions documented and referenced to tender instruction clause

TASK 1.3.2-06

Compile Corrected Tender Ranking Schedule

Description
Compile the final ranking schedule listing all tenderers in ascending order of corrected tender total. Include: tenderer name, original read-out total, number of arithmetic errors found, error percentage, corrected total, qualification status (pass/fail), and B-BBEE level for preference scoring. Clearly mark disqualified tenderers with reasons. This schedule forms the primary input to the Comparative Analysis sub-process (D1.3-SP-003).
Role
QS
Inputs
  • Final corrected totals (TASK 1.3.2-05)
  • Qualification matrix (TASK 1.3.2-03)
  • B-BBEE levels per tenderer
Outputs
  • Corrected tender ranking schedule
  • Evaluation register (complete record)
Tools / Templates
Tender Ranking Schedule Template; Evaluation Register Template
SLA
0.5 working day

✔ Quality Check: All qualified tenderers ranked, all disqualified tenderers listed with reasons, ranking verified against corrected totals, schedule reviewed and signed by Senior QS
