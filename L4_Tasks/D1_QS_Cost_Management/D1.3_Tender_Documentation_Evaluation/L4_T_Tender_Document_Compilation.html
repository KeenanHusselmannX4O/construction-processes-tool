1. Task Overview

This task set decomposes the Tender Document Compilation sub-process (D1.3-SP-001) into eight granular work instructions. Each task covers one discrete activity required to assemble a complete, regulation-compliant tender package — from confirming the procurement strategy with the Employer through to formal issue of documents to invited tenderers. All tasks align with the CIDB Standard for Uniformity, applicable contract forms (JBCC PBA 6.2 / NEC4 / GCC 2015), PPPFA 2017 requirements, and SACQSP professional guidelines.

2. Detailed Task Breakdown

TASK 1.3.1-01

Confirm Procurement Strategy & CIDB Grading

Description
Meet with Employer / PA to confirm tender procurement route (open, selective, or negotiated) based on project value, complexity, and programme requirements. Verify CIDB contractor grading designation required. Confirm whether PPPFA applies (government-funded projects). Document decisions in a tender strategy memorandum signed by employer.
Role
QS (lead) + Employer / PA
Inputs
  • Approved project value and scope from D1.2 BoQ output
  • Employer procurement preferences and funding source
  • CIDB Register of Projects grading thresholds
  • PPPFA 2017 applicability assessment
Outputs
  • Tender strategy memorandum (signed)
  • CIDB grading designation confirmed
  • PPPFA applicability determination
Tools / Templates
Tender Strategy Memo Template; CIDB grading calculator; PPPFA threshold reference table
SLA
1 working day
Decision Criteria
Open tender required for public-sector contracts above R200k per CIDB regulations; selective tender permitted for private clients; negotiated only where justifiable per SACQSP practice guidelines

✔ Quality Check: Strategy memo signed by Employer, CIDB grading verified against current Register of Projects thresholds

TASK 1.3.1-02

Collate Design Team Deliverables

Description
Request and collate all design team deliverables: architectural drawings, structural drawings, mechanical drawings, electrical drawings, civil drawings, and all corresponding technical specifications. Verify drawing issue status using the drawing register. Confirm all drawings have reached "Issued for Tender" status and carry the correct revision. Cross-reference specification clauses against drawing content.
Role
PA / QS (coordination lead)
Inputs
  • Architectural drawing set + specification
  • Structural engineer drawings + specification
  • Mechanical engineer drawings + specification
  • Electrical engineer drawings + specification
  • Civil/site engineer drawings + specification
  • Drawing register from each discipline
Outputs
  • Complete drawing bundle (all disciplines, "Issued for Tender")
  • Complete specification bundle
  • Master drawing register (consolidated)
Tools / Templates
Drawing Register Template (Excel); Design Deliverable Tracker; Document Management System (DMS)
SLA
3–5 working days

✔ Quality Check: All disciplines represented, all drawings at correct revision, no "Preliminary" status drawings included, drawing register signed off by each discipline lead

TASK 1.3.1-03

Compile BoQ Package (Volume 3: Pricing Document)

Description
Assemble the complete pricing document from finalised D1.2 BoQ outputs. Package must include: Preliminaries (per ASAQS Standard System), Preambles to Trades, Measured Works (all trades), Provisional Sums, Prime Cost amounts, Day Works schedule, and Summary. Ensure BoQ pagination is sequential and section references are consistent with specification clause numbering.
Role
QS
Inputs
  • Finalised BoQ from D1.2 BoQ Production process
  • Preliminaries schedule
  • Trade preambles (per ASAQS Standard System)
  • PS/PC schedules as agreed with design team
Outputs
  • Volume 3: Pricing Document (complete BoQ package)
  • BoQ contents page with page references
Tools / Templates
ASAQS Standard System of Measuring; BoQ compilation template (Excel/CostX); Preliminaries template per JBCC/NEC
SLA
1 working day

✔ Quality Check: BoQ totals reconcile to elemental estimate (within ±2%), Prelims/Preambles complete, all PS/PC items carry allowances approved by PA, pagination continuous

TASK 1.3.1-04

Draft Tender Instructions & Returnable Schedules (Volume 1)

Description
Draft Volume 1 using the CIDB Standard for Uniformity template as baseline. Include: formal invitation to tender, submission requirements (number of copies, format, delivery method), closing date and time, tender validity period, evaluation methodology and criteria, and all returnable schedules (Form of Offer, authority of signatory, construction programme, proposed subcontractors, plant & equipment, experience schedule, B-BBEE certificate, tax compliance).
Role
QS
Inputs
  • CIDB Standard for Uniformity — Scope of Works template
  • Tender strategy memo (TASK 1.3.1-01)
  • Employer evaluation criteria preferences
  • PPPFA scoring model (if applicable)
Outputs
  • Volume 1: Tender Instructions
  • Full set of returnable schedules (blank)
  • Evaluation criteria weighting matrix
Tools / Templates
CIDB Standard for Uniformity template; PPPFA evaluation scorecard; Returnable Schedules Pack template
SLA
2 working days
Decision Criteria
PPPFA 80/20 formula for contracts ≤ R50m; 90/10 for contracts > R50m. Evaluation criteria weightings must be disclosed in tender instructions per PPPFA regulations

✔ Quality Check: All CIDB mandatory returnable schedules present, evaluation criteria clearly stated and weighted, closing date allows minimum 21 calendar days tender period per CIDB guidelines

TASK 1.3.1-05

Compile Contract Conditions (Volume 2: Contract Data)

Description
Prepare Volume 2 based on the chosen form of contract. For JBCC PBA 6.2: complete Contract Data, Contract Instructions, and any special conditions. For NEC4: complete Contract Data Part One (Employer data) and Part Two (Contractor data). For GCC 2015: complete Schedule of Variables. Include penalty clauses, insurance requirements, performance guarantee provisions, dispute resolution mechanisms, and retention provisions per SA construction industry norms.
Role
QS + Legal Advisor
Inputs
  • Selected contract form (JBCC PBA 6.2 / NEC4 ECC / GCC 2015)
  • Employer-specific special conditions
  • Insurance requirements schedule
  • Programme and penalty provisions
Outputs
  • Volume 2: Contract Data (complete)
  • Special conditions schedule (if any)
  • Insurance requirements annexure
Tools / Templates
JBCC PBA 6.2 Contract Data template; NEC4 Contract Data template; GCC 2015 Schedule of Variables; Legal review checklist
SLA
1–2 working days
Decision Criteria
Contract form selection confirmed in tender strategy memo; special conditions must not contradict standard form provisions; penalty rate typically 0.5%–1.0% of contract value per calendar day

✔ Quality Check: All contract data fields populated, legal advisor review completed, no conflicts between special conditions and standard form, penalty and retention provisions confirmed by Employer

TASK 1.3.1-06

Assemble Complete Tender Package & Cross-Reference Check

Description
Assemble all six volumes into the complete tender document package. Generate master table of contents. Perform systematic cross-reference check: verify BoQ item descriptions match specification clauses, drawing references in BoQ match actual drawing numbers, Prelims items align with contract conditions, and returnable schedules are complete. Ensure consistent formatting, headers, and page numbering throughout.
Role
QS
Inputs
  • Volume 1: Tender Instructions (TASK 1.3.1-04)
  • Volume 2: Contract Data (TASK 1.3.1-05)
  • Volume 3: Pricing Document (TASK 1.3.1-03)
  • Volume 4: Specifications (TASK 1.3.1-02)
  • Volume 5: Drawings (TASK 1.3.1-02)
  • Volume 6: Addenda placeholder
Outputs
  • Complete tender document package (all volumes)
  • Master table of contents
  • Cross-reference verification log
Tools / Templates
Tender Package Assembly Checklist; Cross-Reference Verification Template; Document formatting style guide
SLA
1 working day

✔ Quality Check: All 6 volumes present and correctly sequenced, cross-reference log shows zero unresolved conflicts, master table of contents accurate, page numbering continuous

TASK 1.3.1-07

QA Review — Completeness & Consistency Check

Description
Independent Senior QS reviews the complete tender package against the QA deliverable checklist. Verify: all mandatory CIDB documents included, evaluation criteria match tender instructions, BoQ arithmetic is correct at summary level, contract data is consistent with tender instructions, all returnable schedules are present and correctly formatted. Record any non-conformances and return to QS for correction before PA approval.
Role
Senior QS (independent reviewer)
Inputs
  • Complete tender package (TASK 1.3.1-06)
  • QA Deliverable Checklist
  • CIDB mandatory document requirements list
Outputs
  • QA sign-off certificate (or non-conformance report)
  • Completed QA checklist
Tools / Templates
QA Deliverable Checklist Template; Non-Conformance Report Template; CIDB compliance matrix
SLA
1 working day

✔ Quality Check: QA checklist 100% complete with zero outstanding non-conformances, Senior QS sign-off obtained, all corrections verified and closed out

TASK 1.3.1-08

Principal Agent Approval & Issue to Tenderers

Description
Submit QA-approved tender package to the Principal Agent for final review and authorisation to issue. PA confirms: package is complete, tender period is adequate, tenderer list is confirmed (for selective tenders), and issue method is agreed (electronic via platform, USB, or hard copy). Upon PA sign-off, issue tender documents to all invited tenderers simultaneously. Record issue date, tenderer details, and method of delivery in the tender register. Send formal invitation letters.
Role
PA (approval) / QS (issue)
Inputs
  • QA-approved tender package (TASK 1.3.1-07)
  • Confirmed tenderer list (selective tenders)
  • PA approval form
Outputs
  • Approved tender documents (PA signed)
  • Tender register (issue log)
  • Formal invitation letters to tenderers
  • Proof of delivery / download confirmations
Tools / Templates
PA Approval Form; Tender Register Template; Invitation Letter Template; Electronic tender platform (if applicable)
SLA
1–2 working days
Decision Criteria
PA must confirm minimum 21 calendar day tender period from issue date; all tenderers must receive documents simultaneously to ensure fairness; any late additions to tenderer list require PA written approval

✔ Quality Check: PA approval signature obtained, all tenderers received documents on same date, tender register complete with delivery confirmation, tender period ≥ 21 calendar days
